Dataset design

Alan Health Dataset Model

This page explains how the data behind the profitability cockpit should be structured: source tables, joins, example rows and the final reporting table. The central grain is one covered member for one reporting month.

1Core Grain

The reporting should be built from member-month exposure, not raw member count.

Member-month spine
One row means

one member x one month x one contract x one product.

Why it matters

A member covered for half of April counts as 0.5 exposure, not 1.0.

Main join key

member_id, contract_id, product_id, reporting_month.

Dashboard output

All profitability metrics are aggregated from this spine into reporting segments.
